# Resume Analyzer API

Небольшой сервер на Node.js/Express, который принимает резюме (файл `.pdf` / `.docx` / `.txt` или просто текст),
отправляет его в Claude через Anthropic API и возвращает структурированный отчёт: оценку, сильные стороны,
слабые места и рекомендации.

## Локальный запуск

```bash
npm install
cp .env.example .env
# впишите свой ANTHROPIC_API_KEY в .env
npm start
```

Сервер поднимется на `http://localhost:3000`. Проверить эндпоинт:

```bash
curl -X POST http://localhost:3000/analyze \
  -H "Content-Type: application/json" \
  -d '{"text":"текст вашего резюме..."}'
```

## Деплой на Railway (так же, как вы уже разворачивали Telegram-бота)

1. Создайте новый репозиторий на GitHub и запушьте туда файлы из этой папки
   (`server.js`, `package.json`, `.env.example`, `.gitignore`).
2. В Railway: **New Project → Deploy from GitHub repo** → выберите репозиторий.
3. В настройках проекта (**Variables**) добавьте переменные окружения:
   - `ANTHROPIC_API_KEY` — ваш ключ из консоли Anthropic
   - `ALLOWED_ORIGIN` — домен(ы) вашего сайта на Тильде, например
     `https://kadrovye-rezervy.ru` (если несколько — через запятую, без пробелов)
4. Railway сам определит Node.js-проект и запустит `npm start`.
5. В разделе **Settings → Networking** нажмите **Generate Domain** — получите публичный URL вида
   `https://your-app.up.railway.app`.
6. Вставьте этот URL в переменную `API_URL` во фронтенд-блоке (см. `frontend-snippet.html` рядом
   или сообщение в чате с кодом для Тильды).

## Сбор лидов (имя + телефон)

Перед анализом фронтенд-блок просит указать имя и телефон. Сервер:

- требует оба поля (`name`, `phone`) — без них вернёт ошибку 400;
- логирует каждую заявку в консоль (`console.log`), это видно в логах Railway;
- если заполнены `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` — присылает уведомление о новом лиде вам в Telegram
  (можно использовать того же бота, что вы уже разворачивали для резюме).

Если нужно сохранять лиды в базу (а не только в логи/Telegram) — можно добавить, например, Google Sheets API
или простую БД (Postgres на Railway тоже есть в один клик). Скажите, если это понадобится — помогу подключить.

## Важно про безопасность и лимиты

- Ключ `ANTHROPIC_API_KEY` **никогда** не должен попадать в код фронтенда/Тильды — только в переменные
  окружения на сервере. Именно поэтому нужен отдельный бэкенд, а не прямой вызов API из браузера.
- `ALLOWED_ORIGIN` ограничивает, с каких доменов разрешены запросы (CORS) — не оставляйте `*` в проде,
  укажите точный домен сайта.
- Файл ограничен 8 МБ (`multer limits`). При необходимости увеличьте в `server.js`.
- Каждый запрос — это один вызов Anthropic API, то есть расходует токены/оплату по вашему аккаунту.
  Если планируете много трафика, стоит добавить rate-limiting (например, пакет `express-rate-limit`).
- `pdf-parse` иногда даёт "грязный" текст на сложных PDF (с колонками, картинками вместо текста) —
  для большинства резюме этого достаточно, но не 100% случаев.
